# 🏯 栖居之道 —— 中国古代民居气候适应性智慧图谱

> **中国大学生计算机设计大赛参赛作品** | 云南大学 信息学院

**栖居之道**是一个面向中国古代传统民居的气候适应性数据可视化平台。系统收录了全国 **25 种**典型传统民居，涵盖 **5 大气候区**、**6 个历史时期**，从保温、隔热、通风、防潮、采光五个维度，以交互式图表直观展示古人"因地制宜"的建筑智慧。

## 🎯 项目概述

### 项目背景

中国传统民居在数千年的演变中，形成了丰富多样的建筑形态。从陕北窑洞到福建土楼，从北京四合院到傣族竹楼，每一种民居都蕴含着古人应对当地气候环境的独特智慧。本项目旨在通过数据可视化的方式，系统性地呈现这些建筑智慧，让用户直观地理解不同气候条件下民居的适应性特征。

### 核心功能

| 功能 | 说明 |
|------|------|
| 📊 **数据可视化** | 7 种图表类型（地图、柱状图、雷达图、散点图、玫瑰图、箱线图、排名图） |
| 🏠 **民居数据库** | 25 种民居的详细参数、评分、气候数据 |
| 🔬 **交互实验室** | 自定义建筑参数，实时评分与优化建议 |
| 🛠️ **工具箱** | 窑洞模拟、南北对比、能耗估算、学习资源 |
| 👤 **用户系统** | 注册登录、收藏管理、成就系统 |

## 🚀 快速开始

### 方式一：直接打开（静态浏览）

由于项目为纯前端应用，可直接在浏览器中打开任意 HTML 文件进行浏览：

```bash
# 在项目根目录下
start index.html
```

> ⚠️ **注意**：直接打开时，Firebase 用户认证和收藏功能需要配置有效的 Firebase 项目密钥才能使用。数据可视化功能（图表、地图等）可正常浏览。

### 方式二：使用本地服务器（推荐）

推荐使用本地 HTTP 服务器以获得最佳体验（避免跨域问题）：

```bash
# 使用 Python（推荐）
python -m http.server 8080

# 或使用 Node.js
npx http-server . -p 8080
```

然后在浏览器中访问 `http://localhost:8080`。

### 方式三：部署到 Firebase Hosting

```bash
# 安装 Firebase CLI
npm install -g firebase-tools

# 登录并初始化
firebase login
firebase init hosting

# 部署
firebase deploy --only hosting
```

|------|------|------|

| 🏆 **综合排名柱状图** | 中右 | 所有民居的综合评分排名 |
| 🌹 **气候适应性玫瑰图** | 中下左 | 各气候区在五个维度的平均得分对比 |

2. **查看详情**：点击地图标记或图表中的数据点，右侧面板显示民居详情
3. **高亮对比**：点击图表中的柱子/数据点，相关民居在地图和其他图表中同步高亮
4. **导出图表**：点击"导出图片"，在弹出的对话框中选择要导出的图表

### 2. 民居库

**路径**: [`houses.html`](houses.html)

民居库是系统的核心数据浏览页面，以卡片网格形式展示所有民居。

#### 左侧边栏
- **时间轴筛选**：按历史时期筛选
- **气候区筛选**：按气候区筛选
- **快速筛选**：一键筛选"保温最佳"、"通风最佳"、"防潮最佳"、"墙体最厚"

#### 主内容区

**搜索与排序栏**：
- 🔍 **搜索框**：按民居名称搜索
- 📋 **排序下拉框**：按名称、评分、墙体厚度、温度、降水量排序

**图表区**：
- 📊 **评分柱状图**：当前筛选结果中各民居的综合评分对比
- 🎯 **雷达图**：选中民居的五维评分雷达图

**民居卡片网格**：
- 每张卡片显示：民居名称、气候区标签、时期标签、综合评分、墙体厚度、年均温
- 点击卡片可查看详情

**排名列表**：
- 完整排名标签页：显示所有民居的完整排名列表（墙体厚度、温度、降水量、综合评分）

#### 右侧边栏
- **民居详情**：选中民居的完整信息，包括：
  - 基本参数：墙体厚度、窗墙比、屋顶坡度
  - 气候条件：年均温、年降水量、湿度、日照时数
  - 建筑智慧：墙体材料、屋顶形式、保温策略、通风策略、防潮策略
  - 五维评分雷达图
- **推荐阅读**：相同气候区的其他民居推荐

#### 交互操作

1. **筛选与搜索**：组合使用时间轴、气候区、快速筛选和搜索框，精确定位目标民居
2. **排序切换**：点击排名标签页切换不同维度的排名
3. **查看详情**：点击民居卡片或排名列表中的条目，右侧面板更新详情
4. **图表联动**：点击柱状图中的柱子，对应民居高亮显示

### 4. 实验室

**路径**: [`lab.html`](lab.html)

实验室是一个交互式建筑设计模拟器，让用户亲手"设计"传统民居并获取评分反馈。

#### 操作流程

**第一步：选择气候场景**

三种预设气候场景：
| 场景 | 气候特点 | 典型地区 |
|------|----------|----------|
| ❄️ **干冷华北** | 冬季寒冷干燥，夏季炎热 | 北京、河北 |
| 💧 **湿冷江南** | 冬季湿冷，夏季闷热多雨 | 江苏、浙江 |
| 🌴 **湿热岭南** | 全年高温多雨，湿度大 | 广东、福建 |

**第二步：调整建筑参数**

| 参数 | 范围 | 说明 |
|------|------|------|
| 🪟 **窗墙比** | 0.05 ~ 0.45 | 窗户面积与墙面面积的比例 |
| 🧱 **墙体厚度** | 50 ~ 1500 mm | 外墙厚度 |
| ⬜ **院落高宽比** | 0.2 ~ 2.0 | 院落高度与宽度的比值 |
| 🧭 **建筑朝向** | 南 / 东南 / 东 / 西南 | 建筑主立面朝向 |

**第三步：查看评分与反馈**

- **实时评分**：系统根据参数自动计算五个维度的得分（通风、保温、防潮、遮阳、材料适应性）
- **总分显示**：综合评分实时更新
- **剖面图**：Canvas 绘制的建筑剖面示意图，随参数调整实时更新
- **雷达图对比**：用户设计（橙色）与参考民居（蓝色）的雷达图对比
- **优化建议**：系统根据评分结果生成具体的改进建议

#### 交互操作

1. **选择气候场景**：点击气候选项按钮切换场景
2. **调整参数**：拖动滑块或点击朝向选项，所有图表和评分实时更新
3. **查看策略提示**：右侧显示当前气候场景的推荐建筑策略
4. **对比参考**：雷达图直观对比用户设计与真实民居的差异

### 5. 工具箱

**路径**: [`tools.html`](tools.html)

工具箱提供四个实用的分析工具。

#### 工具一：🏔️ 窑洞模拟

模拟陕北窑洞在不同季节的室内外温度变化。

- **季节选择**：春、夏、秋、冬
- **墙体厚度调节**：拖动滑块调整墙体厚度（200 ~ 2000 mm）
- **民居选择**：切换不同民居查看数据
- **结果显示**：
  - 室内温度 vs 室外温度对比
  - 温度波动幅度
  - 智慧解读：说明该民居的气候适应性原理

#### 工具二：🔄 南北对比

对比北方与南方典型民居的建筑特征差异。

- **北方民居选择**：从北方民居列表中选择
- **南方民居选择**：从南方民居列表中选择
- **对比结果**：
  - 柱状图对比：墙体厚度、窗墙比、屋顶坡度
  - 雷达图对比：五维评分
  - 差异分析文字说明

#### 工具三：💰 能耗估算

估算传统民居与现代建筑的能耗成本对比。

- **传统民居选择**：选择一种传统民居
- **现代建筑参数**：输入现代建筑的能耗数据
- **计算年限**：选择计算周期（10 / 20 / 30 年）
- **计算结果**：
  - 传统民居年能耗成本
  - 现代建筑年能耗成本
  - 长期节省金额
  - 节能百分比

#### 工具四：📚 学习资源

提供传统建筑相关的学习资料列表，包括：
- 中国传统建筑文化
- 气候适应性设计原理
- 各地民居特色介绍
- 相关学术论文与书籍推荐

## 🚢 部署指南

### 配置 Firebase

1. 访问 [Firebase Console](https://console.firebase.google.com/) 创建项目
2. 启用 **Authentication**（邮箱/密码登录方式）
3. 启用 **Cloud Firestore**（创建数据库）
4. 在项目设置中获取 Firebase 配置信息
5. 修改 [`js/shared.js`](js/shared.js) 中的 `firebaseConfig` 对象：

```javascript
const firebaseConfig = {
    apiKey: "your-api-key",
    authDomain: "your-project.firebaseapp.com",
    projectId: "your-project-id",
    storageBucket: "your-project.appspot.com",
    messagingSenderId: "your-sender-id",
    appId: "your-app-id"
};
```

### Firestore 数据结构

```
users/
  └── {uid}/
      ├── favorites: ["陕北窑洞", "北京四合院", ...]
      ├── username: "用户昵称"
      └── settings: { language: "zh", theme: "light" }
```

## 💻 开发指南

### 数据扩展

如需添加新的民居数据，编辑 [`data/house.json`](data/house.json) 或直接修改 [`js/shared.js`](js/shared.js) 中的 `allHousesData` 数组。

数据格式示例：

```javascript
{
    name: "民居名称",
    coords: [经度, 纬度],
    climate: "气候区",
    period: "历史时期",
    temp: 年均温,
    precip: 年降水量,
    wall: 墙体厚度(mm),
    ratio: 窗墙比,
    pitch: 屋顶坡度(°),
    scores: {
        保温: 评分(1-10),
        隔热: 评分(1-10),
        通风: 评分(1-10),
        防潮: 评分(1-10),
        采光: 评分(1-10)
    },
    wallMat: "墙体材料",
    roof: "屋顶形式",
    courtyard: "院落布局",
    insulation: "隔热策略",
    heatKeep: "保温策略",
    ventilation: "通风策略",
    moisture: "防潮策略",
    desc: "建筑描述"
}
```

### 添加新页面

1. 在 HTML 中引入 [`css/layout.css`](css/layout.css) 和 [`js/shared.js`](js/shared.js)
2. 参考现有页面的导航栏和布局结构
3. 使用 `allHouses` 全局变量获取民居数据

### 自定义样式

项目使用 CSS 变量进行主题管理，在 [`css/layout.css`](css/layout.css) 的 `:root` 中定义：

```css
:root {
    --primary: #C0392B;      /* 主色（朱红） */
    --secondary: #6A994E;    /* 辅助色（青绿） */
    --accent: #D4AF37;       /* 强调色（金色） */
    --bg-light: #FDF8F0;     /* 背景色（米白） */
    --text-dark: #2C3E2F;    /* 文字色（深绿） */
}
```

## ❓ 常见问题

### Q: 为什么地图不显示？

A: 确保网络连接正常，ECharts 中国地图插件需要从 CDN 加载地图数据。如加载失败，可尝试更换网络环境或使用代理。

### Q: 登录/注册功能无法使用？

A: 需要配置有效的 Firebase 项目密钥。请参考[配置 Firebase](#配置-firebase) 部分进行设置。

### Q: 图表数据不更新？

A: 检查浏览器控制台是否有 JavaScript 错误。部分图表需要 ECharts 库正确加载。

### Q: 如何添加更多民居数据？

A: 编辑 [`js/shared.js`](js/shared.js) 中的 `allHousesData` 数组，按照数据格式添加新条目即可。

### Q: 页面布局错乱？

A: 本项目针对 1920×1080 及以上分辨率优化。如在小屏幕设备上访问，建议使用桌面端浏览器并放大/缩小页面。
